# Retention target ("40% retention")

The headline retention number everyone cites — with three different definitions.

**Current: the 40% threshold is agreed; the measurement window is not.** Treat any "40% retention"
claim as ambiguous until a window is attached.

## The conflict (metric drift D)
| Source | Definition |
| --- | --- |
| PRD §Success (2026-01-22) | **30-day** retention ≥ 40% (warning < 25%), measured weekly — primary product KPI |
| OKR (2026-03-05) | **7-day** retention ≥ 40% — north-star metric |
| Roadmap (2026-03-30) | North-star = 7-day retention (defers to OKR) |
| GTM (2026-04-25) | bare **"40% retention"** in external decks — no window |

Per `CLAUDE.md` §4: metrics with the same name but different definitions are a **contradiction,
not a duplicate**. 30-day ≥40% and 7-day ≥40% are very different bars.

## Open issues
- **Needs one agreed definition** of "the retention target," and external decks must stop citing
  a window-less "40% retention."
